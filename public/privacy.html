1. 收集范围

我们收集的信息类型

基础账户信息

  • 商家店铺名称、联系人姓名
  • 手机号码、电子邮箱地址
  • 淘宝店铺ID和认证信息
  • 营业执照等经营资质文件

业务相关数据

  • 商品信息：商品标题、描述、属性、价格
  • 订单数据：订单号、商品详情、收货地址、联系方式
  • 客服记录：买家咨询内容、客服回复记录
  • 物流信息：快递单号、物流轨迹、配送状态
  • 评价信息：买家评价、店铺评分数据

技术日志信息

  • API调用日志、错误记录
  • 系统访问日志、操作轨迹
  • 设备标识、IP地址信息
  • 浏览器类型、操作系统信息

2. 使用目的

主要用途说明

核心服务提供

  • 智能客服分流：分析买家咨询意图，匹配合适的客服或自动回复
  • 商品内容优化：基于商品信息生成优化后的标题和描述
  • 违禁词检测：扫描和识别违规词汇，保障合规性
  • 数据分析：生成客服效率和转化率分析报告

服务质量保障

  • 监控和优化系统性能
  • 故障排查和问题解决
  • 用户体验改进和产品迭代
  • 新功能开发和测试

3. 处理方式

数据处理原则

最小必要原则

仅处理提供服务所必需的数据，不收集与业务无关的信息。对敏感数据进行分类分级管理，确保最小化处理范围。

自动化处理流程

  • 实时分析：买家咨询内容实时解析和分类
  • 智能生成：基于商品信息自动生成优化内容
  • 批量处理：支持大量商品信息的批量操作
  • 质量检测：自动化违禁词检测和合规校验

人工审核机制

对自动化处理结果进行抽样人工复核，特别是涉及争议或异常情况的处理，确保服务质量。

4. 共享与第三方

数据共享原则

不向第三方出售数据

我们承诺不会向第三方出售、交易或租赁您的个人信息和业务数据。

必要服务共享

在以下必要情况下，可能与第三方共享数据：

淘宝开放平台

  • 基于您的授权访问淘宝API
  • 严格按照淘宝平台规则使用数据
  • 不超出授权范围处理信息

法定披露义务

根据法律法规要求，在以下情况下可能披露数据：

  • 执法部门依法调查
  • 司法程序需要
  • 保护公司合法权益

5. 跨境情况

数据存储位置

  • 主要存储：中国大陆境内服务器
  • 备份存储：同城灾备机房
  • 访问控制：仅限授权人员境内访问

跨境访问说明

目前服务数据存储和处理均在境内的AWS中国区域或阿里云国内机房，不涉及跨境数据传输。

6. 保留期限

默认保留策略

日志数据保留：180天

  • API调用日志：保留180天后自动删除
  • 系统访问日志：保留180天后自动删除
  • 错误记录：保留180天后自动删除
  • 性能监控数据：保留180天后自动删除

业务数据保留：服务期限内

  • 商品信息：随服务持续保留，可随时删除
  • 订单数据：随服务持续保留，可随时删除
  • 客服记录：随服务持续保留，可随时删除

数据删除机制

  • 自动删除：日志类数据按策略自动清理
  • 用户申请删除：业务数据支持随时申请删除
  • 服务终止清理：停止服务后30天内完成数据清理

7. 数据主体权利

用户权利保障

  • 知情权：随时查看我们收集的个人信息
  • 访问权：获取个人数据的副本
  • 更正权：更新或更正不准确的个人信息
  • 删除权（被遗忘权）：申请删除个人数据
  • 限制处理权：限制特定数据的使用范围
  • 数据导出权：获取结构化的数据副本

8. 退出与删除

授权解绑流程

淘宝平台授权解绑

  1. 登录淘宝开放平台
  2. 进入应用管理页面
  3. 点击"解除授权"按钮
  4. 确认解绑操作

数据导出流程

  1. 发送邮件至：privacy@taobao-context-service.com
  2. 提供身份验证信息
  3. 明确数据导出范围和格式
  4. 我们在7个工作日内完成处理

数据导出申请

邮箱：privacy@taobao-context-service.com

处理时间：7个工作日内

导出内容：账户信息、商品数据、订单记录、客服记录、使用日志

彻底删除流程

  1. 提供书面删除申请
  2. 身份和权限验证
  3. 确认删除范围和后果
  4. 签署删除确认书

9. 儿童条款

儿童数据保护

我们的服务主要面向年满18岁的商家和成年人。我们不会故意收集未满14岁儿童的个人信息。

如果发现14岁以下儿童的个人信息被不慎收集，我们将：

  1. 立即停止相关数据处理活动
  2. 在3个工作日内删除所有相关数据
  3. 不向第三方共享或泄露该信息
  4. 记录处理过程以备监管检查

10. 信息安全措施

技术安全保障

  • 数据加密：采用TLS 1.3协议加密传输，AES-256加密存储
  • 访问控制：多因素身份验证，基于角色的访问控制
  • 系统安全：防火墙保护，7×24小时实时监控
  • 灾备机制：多地备份和快速恢复能力

应急响应

  • 发现机制：7×24小时安全监控中心
  • 响应时间：重大安全事件1小时内响应
  • 恢复目标：核心业务4小时内恢复

11. 变更通知

政策更新机制

  • 重大变更：提前30天邮件通知 + 服务内弹窗
  • 一般变更：提前7天邮件通知 + 官网公告
  • 紧急变更：及时通知 + 说明变更原因

用户权利保障

用户有权拒绝接受的变更，拒绝接受的用户可申请退款，我们提供数据导出和删除服务。
